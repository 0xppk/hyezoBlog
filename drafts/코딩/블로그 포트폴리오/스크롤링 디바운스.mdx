### 스크롤에 반응하는 네비바

스크롤이 내려가면 네비바가 보이도록 만들었다.  
하지만 이 과정에서 콘솔창을 찍어보면 이게 맞나? 불안함 감정이 절로 든다.  
스크롤 한번에도 숫자들이 비명지르듯 솟구쳐 오르기 때문이다.  
이때 효율을 개선할 수 있는 방법이 바로 `throttle`이다.

### Throttle vs Debounce

쓰로틀은 함수가 지정된 시간에 한번만 호출되도록 제약을 주는 함수 또는 개념이다.  
비슷하게 디바운스가 있는데, 디바운스는 (연속된 동작 가운데) 마지막 실행이 멈춘 뒤 함수를 한번 실행시킨다는 점에서 차이가 있다. 이때 마지막이라 함은 지정시간 이상의 휴지기를 가진 경우를 의미한다. 
<br/>
가령 이런 예를 들어보자.  
머리 위로 팔을 3시 방향에서 9시 방향으로 일정하게 휘두르려고 한다. 만약 이때 딜레이 간격을 3시간 간격으로 잡았다면 `throttle`은 12시에서 한번, 9시에 도착했을 때 한번, 이렇게 두번 실행될 것이다. 반면 `debounce`는 9시에 도착해 세시간이 지난 뒤 명령을 한번 내뱉고 말 뿐이다. (물론 12시에 도착해 세시간을 쉬고, 9시에 도착해 세시간을 쉬면 똑같이 12시와 9시가 찍힐 것이다.) 
<br/>
이해가 됐을까? 개념을 잡았다면 쓰면 된다. <q>lodash</q>에서는 이를 아주 쉽게 쓸 수 있게 함수로 제공해 주고 있다. 우리는 심장소리처럼 스크롤바를 경청할 예정이오니 디바운스가 아닌 쓰로틀을 걸어준다

```tsx Layout.tsx
⌇
import { useMemo, useEffect, useRef, useState } from "react";
import { throttle } from "lodash";

const [navShow, setNavShow] = useState(false);
  const beforeScrollY = useRef(0);
  const scrollSensor = useMemo(
    () =>
      throttle(() => {
        const currentScrollY = globalThis.scrollY;
        if (currentScrollY > beforeScrollY.current && currentScrollY > 60) setNavShow(true);
        else setNavShow(false);
        beforeScrollY.current = currentScrollY;
      }, 300),
    [beforeScrollY],
  );

  useEffect(() => {
    globalThis.addEventListener("scroll", scrollSensor);
    return () => globalThis.removeEventListener("scroll", scrollSensor);
  });
```


### 간단설명
`globalThis` : ES11에 추가된 전역객체다. 통상 전역객체하면 `window`, `this`를 많이 썼는데, 환경에 따라 `self`, `global` 같은 녀석들을 써야했다고 한다. 하지만 globalThis는 환경과 무관하게 전역객체에 접근함으로써 이 혼돈을 정리한다. 코알못인 나에겐 아주 기특한 녀석으로 보였기 때문에 window 대신 애용 중이다.  
<br/>
`useRef()` : 원래 let으로 스크롤값을 저장했는데, 빌드로그에 useRef를 쓰라고 경고가 날아왔다. 이 역시 일종의 변수, 보관통이라고 보면 될 텐데 다른 변수들과의 차이점을 짚어보면
   - vs *useState* : state를 업데이트하면 component 또한 리렌더된다. 하지만 useRef에 저장하는 데이터는 리렌더를 초래하지 않는다.
   - vs *변수* : component가 리렌더되면 변수는 초기화된다. 하지만 useRef는? 그대로 보존된다.  

useRef를 쓰면 좀더 안전하고 효율적으로 값을 저장할 수 있을 것처럼 보인다. 데이터를 바꿀 때는 .current를 붙여서 사용한다.
<br/>
`useMemo()` : 불필요한 리렌더링을 방지해주는 훅이다. 의존성으로 입력한 값이 변화할 때만 움직이는 엉덩이 무거운 콜백 함수다. 메모리를 사용해서 계산값 붙들고 있기 때문에 적절히 쓰면 성능 향상을 기대할 수 있다고 한다. 어떻게 보면 useEffect와도, useCallback과도 겹치는 부분이 있는데 그건 다음에 정리해보려고 한다.  
결론적으로 `throttle`에 의해 300ms마다 스크롤값이 바뀜으로써 지속적으로 스크롤 위치를 마킹하게 된다.  
(근데 값을 붙드는 건 useRef가, 갱신 타이밍은 throttle로 재고 있어서, useMemo는 빼도 무방할 듯싶다.)

### 유니언 타입을 확장하기
아래 `Origin` 타입을 가공해 `After`와 같은 타입을 만들어내는 것이 오늘의 미션!

<CH.Code>
```ts type.ts
export type Origin = 
  | {
    type: "user";
  }
  | {
    type: "post";
  }
  | {
    type: "comment";
  }

type After = 
  | {
    type: "user";
    userId: string;
  }
  | {
    type: "post";
    postId: string;
  }
  | {
    type: "comment";
    commentId: string;
  }
```
---
```ts example.ts
@ts-expect-error
const result: MyType = {
  type: "user",
  userId: "1"
}
```
</CH.Code>

### 접근
<CH.Spotlight>
```ts index.ts
type MyType = {
  
}
```
---
- 우선 `Origin`의 유니언 타입에서 `type의 값`을 가져온다.  
키 인덱스 시그니처를 이용한다.
```ts index.ts focus=2:4
type MyType = {
  [OriginTypes in Origin["type"]]: {
    type: OriginTypes;
  }
}
```
---
- 디스트럭션을 사용해 바깥 오브젝트를 풀어준다.
```ts index.ts focus=2:5 mark=5[2:17]
type MyType = {
  [OriginTypes in Origin["type"]]: {
    type: OriginTypes;
  }
}[Origin["type"]]  // user | post | comment 
```
---
- 그러면 이런 결과를 가지게 될 텐데
```ts result.ts
type MyType =
  | {
    type: "user" 
  }
  | { 
    type: "post"
  }
  | { 
    type: "comment"
  }
```
---
- 데이터 추가를 위해 `Record`를 쓴다.  
`Record`는 인덱스 시그니처처럼 오브젝트의 프로퍼티(키 밸류 한쌍)를 다이나믹하게 터주는 효과를 가지고 있다.  
이 경우엔 키값을 리터럴 타입으로, 배정받은 `type`값을 할당하면서 새로운 프로퍼티로 합류시켰다.
```ts index.ts focus=4
type MyType = {
  [OriginTypes in Origin["type"]]: {
    type: OriginTypes;
  } & Record<`${OriginTypes}Id`, string>
}[Origin["type"]]
```

</CH.Spotlight>

### 이번 포스트를 통해 새로 알거나 정리된 것
1. 확장의 두 종류
- `extends`를 통한 상속과 `&(interSection)`을 통한 결합의 차이  
공통된 뿌리가 있을 땐 상속, 개별적으로 덧대는 상황에서는 인터섹션을 쓴다.  
`extends`가 크게 퉁치고 나가는 느낌이라면 `intersection`은 전술적으로 디테일하게 쓰이는 느낌임.
<br/>
2. 인덱스 시그니처의 쓰임
- `next.js`에서 대괄호로 다이나믹 라우팅을 처리하듯 얘도 그런 역할이다.  
`in` 또는 `in keyof`와 결합해 반복문 돌리듯 다이나믹하게 키를 늘어놓는다. 이런 반복문적인 요소란 코딩에서 웬만하면 꿀처럼 사용되니 딱 잡아놓는 게 좋을 듯ㅎ  
`Record`는 덤이다. 예시로 둘의 유사성을 남겨 놓는다

```ts Record
const Ex1: {
  [keys: string]: string | number
} = {
  id: 1,
  name: "hyezo"
}

const Ex2: Record<string, string | number> = {
  id: 1,
  name: "hyezo"
}
```
<br/>
하지만 인덱스 시그니처는 리터럴 형식을 받지 않는다. 이런 점에서 `Record`가 좀더 확장성 있고 요긴하게 쓰일 듯하다.
